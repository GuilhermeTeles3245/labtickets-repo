# Sprint 04 — Detalhes, Comentários e Fechamento

• **Sprint Goal:** Permitir a interação dentro do chamado (comentários) e a mudança de status até a conclusão.
• **Itens selecionados (Sprint Backlog):** 
  - Página de detalhes do chamado (`/chamado/<id>`).
  - Funcionalidade de adicionar comentário.
  - Ação do técnico para mudar status (Novo -> Andamento -> Concluído).
  - Testes finais e fechamento do MVP.
• **Daily (resumo):** 
  - Dia 1: Layout da página de detalhes.
  - Dia 2: Backend da rota de comentários.
  - Dia 3: Lógica para técnico "assumir" o chamado automaticamente ao mudar para "Em Andamento".
  - Dia 4: Revisão geral da aplicação, testando responsividade no celular.
  - Dia 5: Gravação do vídeo/demo e ajustes no README.
• **Review Final:** O MVP foi demonstrado completo. O Coordenador conseguiu visualizar o sistema fluindo perfeitamente.
• **Retrospective Final:** 
  - Ação 1: O uso de Python/Flask foi a decisão mais acertada para entregar tudo tão rápido. O grupo está pronto para a entrega.
